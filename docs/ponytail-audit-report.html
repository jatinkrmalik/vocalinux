Executive summary

Vocalinux is a real multi-engine, multi-desktop dictation app — much of the surface is platform necessity (IBus / wtype / ydotool / xdotool, pynput + evdev, vosk + whisper.cpp). The bloat is not “too many features for Linux,” it is duplication, coverage-chasing tests, dead alternate implementations, and marketing-site scaffolding that never ships in the app.

Biggest gravity wells: settings_dialog.py (3.4k), recognition_manager.py (3.1k), install.sh (3.9k), tests (~29k, often 2–10× product), and a web SEO page farm (~25 near-identical routes).

High-confidence wins

  • Delete dead deps: pydub, tqdm, python-xlib
  • Delete unused web: lib/api/util.ts, CodeBlock, Button, DictationOverlay
  • Delete custom AboutDialog class (tray already uses Gtk.AboutDialog)
  • Collapse recognition_manager test clones (~3.5–4.5k tests)
  • Delete test_integration_branches.py kitchen sink (~1.5k)

Product YAGNI (decide deliberately)

  • Third local engine: OpenAI Whisper (torch) alongside whisper.cpp
  • Advanced whisper.cpp knobs UI (~300 lines settings)
  • install.sh as full package manager vs pip-first bootstrap
  • SEO keyword page farm vs content-data + one template

What to keep (not over-engineering)

  • Multiple injection strategies (IBus / wtype / ydotool / xdotool / clipboard) — platform necessity on Linux desktops.
  • Dual keyboard backends (pynput + evdev) — real X11 vs Wayland constraint.
  • vosk + whisper_cpp as engines — different latency/accuracy tradeoffs (third torch engine is the questionable one).
  • Silero VAD module — small and focused (~104 lines).
  • IBus out-of-process socket + PEERCRED — required architecture, not speculative.
  • single_instance via fcntl.flock — correct, lean stdlib approach.
  • Domain-specific test suites for silero, remote API, real Wayland/XKB activation guards, and behavioral text-injector env paths.
  • ActionHandler — small and data-driven.
  • Autostart XDG .desktop — fine as-is.

Suggested cut order (highest ROI first)

  1. Week of free wins: dead Python deps; dead web util.ts + unused components/npm; custom AboutDialog; distro-package-map.yaml; unreachable private recognition helper.
  2. Test diet: delete integration_branches; collapse recognition_manager suites; merge IBus helper tests; hoist fixtures to conftest. Expect CI green with fewer files.
  3. DRY product code: download loops; backend picker; dual settings apply; logging stack; ResourceManager single root.
  4. Product decisions: OpenAI Whisper engine stay/go; advanced knobs UI; install.sh bootstrap-only vs full installer.
  5. Website: data-drive SEO pages (like distro install); demote homepage from client-motion monolith.
Ponytail rule still applies: never cut trust-boundary validation, data-loss handling, security, or accessibility “for fewer lines.” Several IBus retry paths and injection fallbacks exist for real desktop flakiness — simplify structure, don’t delete recovery without evidence.
